JS Behaviours

 Tiny Ruby on Rails plugin for organizing your javascript code sleek and unobtrusive way, inspired by now unmaintained http://ujs4rails.com/ plugin and wonderful lowpro.js library by Dan Webb (http://www.danwebb.net/lowpro , http://github.com/danwrong/low-pro-for-jquery)

Example

 Imagine yet another cool web2.0 application with photos, movies and ratings.

1) Setup controllers:

  class MoviesController < ApplicationController
    behaviours :ratings, :flash_player
  end

  class PhotosController < ApplicationController
    behaviour :ratings
  end

  class Admin::StuffController < ApplicationController
  end

2) Write your behaviours, get library code and organize public/javascript folder:

  public/
    javascripts/
      behaviours/
        admin/
          photos.js      # automagicaly included into Admin::PhotosController views
        movies.js        # automagicaly included into MoviesController views
        photos.js        # automagicaly included into PhotosController views
        comments.js      # included into MoviesController && PhotosController views
        flash_player.js  # included into MoviesController views
      jquery.js
      lowpro.jquery.js
      swfobject.js
      markitup.js
      markitup.markdown.js

3) Specify behaviours dependencies

  // flash_player.js
  // Shared flash player initialization
  // requires swfobject.js
  // ^^^^^^^^^^^^^^^^^^^^^^ -- this line does all magic
  jQuery(function($) {
  ...
  });

  // comments.js
  // WYSIWIG markdown comments
  // requires markitup, markitup.markdown
  // ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ -- this line does all magic
  jQuery(function($) {
  ...
  });

4) Enable magic somewhere in <head>:

  <%= javascript_include_tag :defaults %>
  <%= behaviours_javascript_include_tag %>

Result:

  Included into PhotosController views:
    markitup.js, markitup.markdown.js, photos.js, comments.js

  Included into MoviesController views:
    swfobject.js, markitup.js, markitup.markdown.js, movies.js, comments.js, flash_player.js

Caching

  By default, if action_controller.perform_caching = true, behaviour files concatenated with their library dependencies cached in public/javascripts/behaviours_cache/<controller_name>.js . You can disable it with explicit:

  <%= behaviours_javascript_include_tag :cache => false %>
