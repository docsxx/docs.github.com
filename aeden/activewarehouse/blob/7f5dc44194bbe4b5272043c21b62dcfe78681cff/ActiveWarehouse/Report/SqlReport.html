ActiveWarehouse::Report::AbstractReport

A report which displays the SQL used to generate a tabular report.

Constructor

public initialize(params)

AbstractReport seems to assume it will be mixed into an ActiveRecord::Base subclass — it uses read_attribute (which is one way of accessing model attributes), as well as using attribute accessors that are defined when TableReport inherits from ActiveRecord::Base.

To get around this, we put all the fields into the params hash that is passed into the intialization of the SqlReport. Then we define read_attribute and method_missing to access this hash directly.

If you have a better solution - go for it!
