find_queried

public find_queried(extend, query_params, rules, options = )

Description

Parses the query parameters the client has given in the URL of the HTTP request. With the query parameters, the client may set a limit, an offset, or an ordering to the items in the search result. In addition, the client may limit the output by allowing only certain records that match to specific patterns.

What the client user is allowed to query is defined by specific rules passed to the method as a Hash argument. Query parameters that are not explicitly stated in the rules are silently discarded.

Essentially, the method is a frontend for ActiveRecord::Base#find. The method

  1. parses the query parameters the client has given in the URL for the HTTP request (query_params) against the rules (rules), and
  2. calls find with the parsed options.

Parsing rules

The parsing rules must be given as a Hash; the keys in the hash indicate the parameters that are allowed. The recognized keys are the following:

  • :limit, which uses nil as the value (the same effect as with find).
  • :offset, which uses nil as the value (the same effect as with find).
  • :sort, which determines the ordering. The value is a Hash of ‘parameter value’ => ‘internal table column’ pairs (the same effect as with the :order option of find);
  • :rsort, for reverse sort. Uses the rules of :sort; thus, use nil as the value.
  • :since, which sets a lower timedate limit. The value is either a string naming the database table column that has timestamps (using the type from default settings’ :cast_since_as entry) or a Hash that contains entries :column => ‘table.column’ and :cast_as => ’<sql_timedate_type>’.
  • :until, which sets an upper timedate limit. Used like :since.
  • :patterns, where the value is a Hash containing patterns. The keys in the Hash correspond to additional query parameters and the corresponding values to database table columns. For each pattern, the value is either directly a string, or a Hash containing the entry :column => ‘table.column’. In addition, the Hash may contain optional entries :converter => lambda { |val| <conversion_operation_for_val> } and :operator => ’<sql_pattern_operator>’. :converter expects a block that modifies the input value; if the key is not used, the converter specified in :pattern_converter in the default settings is used. :pattern_operator specifies a custom match operator for the pattern; if the key is not used, the operator specified in :pattern_operator in default settings is used.

If both sort and rsort parameters are given in the URL and both are allowed by the rules, sort is favored over rsort. Unlike with sort and rsort rules (rsort uses the rules of sort), the rules for since and until are independent from each other.

For usage examples, see the example in README and the unit tests that come with the plugin.

Default settings for rules

The default settings for the rules are accessible and modifiable by calling the method searchable_record_settings. The settings are stored as a Hash; the following keys are recognized:

  • :cast_since_as,
  • :cast_until_as,
  • :pattern_operator, and
  • :pattern_converter.

See the parsing rules above how the default settings are used.

Arguments

extend:The same as the first argument to find (such as :all).
query_params:The (unsafe) query parameters from the URL.
rules:The parsing rules as a Hash.
options:Additional options for find, such as :include => [ :an_association ].

Return

The same as with ActiveRecord::Base#find.

def find_queried(extend, query_params, rules, options=;{})
  query_params = {} if query_params.nil?
  query_params = preserve_allowed_query_params(query_params, rules)

  unless query_params.empty?
    parse_offset(options, query_params)
    parse_limit(options, query_params)
    parse_order(options, query_params, rules)
    parse_conditions(options, query_params, rules)
  end

  logger.debug("find_queried: query_params=<<#{query_params.inspect}>>, resulted options=<<#{options.inspect}>>")

  if options.delete(:paginate)
    self.paginate(extend, options)
  else
    self.find(extend, options)
  end
end
