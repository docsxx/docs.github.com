Creating documentation for your project

If you're a project author, you can add a post-commit hook and your documentation will automatically regenerate whenever you push updates to your code. Simply open your project, click on the Admin tab and click on the Service Hooks button. Add the following post-receive URL: 
http://rdoc.info/projects/update
We proudly use Loren Segal's YARD library to generate documentation. This means that you can take advantage of meta @tags alongside regular code. If you'd like to customize the classes that we index, create a .document file in the root directory of your GitHub project. Each line of the file should contain a file name or pattern indicating which files to include (default: lib/**/*.rb).
